# توقعات كأس العالم 2026

نسخة جاهزة للنشر من GitHub إلى Railway، باستخدام Node.js وPostgreSQL.

## النشر على Railway

1. أنشئ مستودعًا جديدًا في GitHub وارفع جميع ملفات المشروع إليه.
2. داخل Railway اختر `New Project` ثم `Deploy from GitHub Repo`.
3. أضف قاعدة بيانات من `Add Service` ثم `Database` ثم `PostgreSQL`.
4. اربط خدمة الموقع بقاعدة البيانات. Railway يضيف `DATABASE_URL` تلقائيًا.
5. أضف المتغير `API_FOOTBALL_KEY` في تبويب `Variables` لخدمة الموقع.
6. انتظر اكتمال النشر، ثم من `Settings > Networking` أنشئ نطاقًا للموقع.

لا تحتاج إلى تنفيذ الجداول يدويًا؛ ينشئها الخادم تلقائيًا عند أول تشغيل.

يجب أن تكون الملفات التالية كلها في جذر المستودع دون مجلدات:

`server.mjs`، `ui.mjs`، `schema.sql`، `package.json`، `railway.json`، `.env.example`، `README.md`.

## متغيرات البيئة

```env
DATABASE_URL=postgresql://...
API_FOOTBALL_KEY=...
NODE_ENV=production
```

## التشغيل المحلي

يتطلب PostgreSQL محليًا:

```bash
npm install
npm start
```

يفتح الخادم على `http://localhost:3000`، أو على قيمة `PORT` التي تحددها.

## أكواد الدخول الأولية

- مشارك: `2026`
- مدير: `ADMIN26`

غيّرها بعد التأكد من عمل النشر بإنشاء مشاركين جدد من لوحة المدير.

## الوظائف

- دخول بكود مشفر لكل مشارك.
- توقع نتائج المباريات وإغلاق التوقع من الخادم عند البداية.
- ترشيحات الأدوار المستقبلية.
- ترتيب وإحصاءات ومقارنة بين المشاركين.
- إنشاء المشاركين واعتماد النتائج واحتساب النقاط.
- مزامنة مباريات كأس العالم عبر API-Football.
- سجل تدقيق للتعديلات الإدارية.

## قواعد النقاط

- 5 نقاط: نتيجة مطابقة.
- 3 نقاط: الفائز وفارق الأهداف صحيحان.
- نقطتان: اتجاه المباراة صحيح.
- 0: اتجاه خاطئ.

تستخدم المسافة عن النتيجة لكسر التعادل بين التوقعات.
